# CIRO — Crisis Intelligence & Response Orchestrator

CIRO is an advanced, production-grade emergency operating system designed for the next generation of smart cities. Built on the **Google Antigravity** multi-agent orchestration framework, CIRO autonomously fuses multi-source signals, analyzes complex situations, and orchestrates real-time tactical responses to stabilize urban crises.

## 🏙️ The Vision: Digital Twin Emergency Ops
CIRO acts as the "City Brain"—a digital twin that monitors every pulse of the city. Even without physical IoT sensors, CIRO leverages synthetic telemetry from social media, weather feeds, and traffic patterns to simulate a city-scale intelligence mesh.

## 🧠 Brain Architecture: Google Antigravity
The system achieves true **Autonomous Agency** through a structured hierarchy of specialized agents developed using the Antigravity pattern.

### The Agentic Mesh
1.  **Supervisor Agent (The Commander)**: Orchestrates the entire lifecycle, manages state escalation, and delegates tasks to specialists.
2.  **Signal Fusion Agent**: Ingests noisy, multilingual (Urdu/English) streams and resolves contradictions to identify core event "signals".
3.  **Credibility Agent**: Uses probabilistic scoring to filter misinformation and spam, ensuring operational focus.
4.  **Classification Agent**: Infers the crisis type (Urban Flooding, Accidents, etc.) and calculates severity and impact radius.
5.  **Resource Agent**: Autonomously bids and allocates the closest, most suitable responders (Ambulances, Rescue Units).
6.  **Simulation Agent (The Strategist)**: Runs parallel "What-If" scenarios to predict the outcome of actions before they are executed.
7.  **Forecast Agent**: Predicts cascading failures (e.g., how a flood at point A gridlocks point B).

## 🚀 Innovation & Agency
-   **Autonomous Decision Loop**: CIRO doesn't just show data; it *acts*. Once confidence crosses a 0.85 threshold, agents trigger tactical rerouting and dispatch without human intervention (Human-in-the-loop is supported for override).
-   **Multilingual Reasoning**: The system understands mixed-language social reports (e.g., Urdu descriptions of local landmarks), translating them into geospatial coordinates.
-   **Strategy Simulation**: By running D3-based outcome simulations, CIRO can discard sub-optimal plans in favor of those that reduce risk scores by the highest percentage.

## 🛠️ Technical Stack
-   **Orchestration**: Google Antigravity (Powered by Gemini)
-   **Real-time Engine**: Node.js/Express with Socket.io for millisecond-latency state synchronization.
-   **Frontend Architecture**: React 19 + Vite + Tailwind CSS 4 + Framer Motion.
-   **Intelligence Layer**: Gemini 3.5 (via `@google/genai`) for complex situational reasoning.
-   **Geospatial Visualization**: High-performance React-Leaflet with dynamic heatmap overlays and satellite topology layers.
-   **Persistence Phase**: Client-side state hydration with server-authoritative simulation tickers.

## 📊 AI Transparency (Decision HUD)
We have implemented a **Reasoning Transparency Panel** that visualizes:
-   **The "Why"**: Natural language explanations generated by the Forensics Agent.
-   **Confidence Heatmaps**: Visual representation of signal density vs. system certainty.
-   **Signal-Incident Graph**: Mapping how disparate social media posts link back to a single root cause.

## 🚀 Deployment (Google Cloud Run)

To deploy CIRO to Google Cloud Run and fix the build errors, follow these exact steps:

1. **Service Settings**:
   - Select "Continuously deploy new revisions from a source repository".
   - Click **SET UP WITH CLOUD BUILD**.
2. **Setup Cloud Build**:
   - Select your GitHub Repository.
   - **Build Configuration**:
     - **Build Type**: Select **Dockerfile** (⚠️ Do NOT select Buildpacks).
     - **Source location**: `/Dockerfile`.
   - Click **Save**.
3. **Advanced Settings**:
   - **Container Port**: Set to `3000` (Google defaults to 8080, make sure this matches or update it to 3000).
   - **Environment Variables**: Add `GEMINI_API_KEY` (if your engine uses it).
4. **Service Name**: Ensure your Service Name does not end with a dash `-` (e.g., use `ciro-engine` instead of `ciro-orchestrator-`).

*The current code includes a specialized `vite.config.ts`, `Dockerfile`, and `.npmrc` specifically tuned to resolve React 19 and Recharts dependency conflicts (`react-is`) during Google Cloud Build.*
